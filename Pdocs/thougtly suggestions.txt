### 🚨 Full-Stack & Architecture (Critical Bugs & Risks)

**1. The "Tags" Data-Type Disconnect (Critical Bug)**

* **Issue:** Tumhare Frontend UI mein tags ka input field ek single text string lag raha hai (jaise placeholder mein hai: `react, javascript, webdev`). Lekin backend validation code mein tum usko array demand kar rahe ho: `.isArray({ min: 1 })`.
* **Impact:** Agar tumhara frontend request bhejte waqt string ko split karke array nahi bana raha (e.g., `tags.split(',').map(t => t.trim())`), toh tumhari har post request backend par **400 Bad Request** de kar fail ho jayegi.
* **Fix:** Frontend payload check karo. Agar frontend se string aa rahi hai, toh backend par custom sanitizer likho jo request body aate hi usko array mein convert kare before validation.

**2. Missing Content Max Length (Security / DoS Risk)**

* **Issue:** `title` ke liye tumne limit lagayi hai (`min: 3, max: 200`), lekin `content` validation mein sirf `.isLength({ min: 10 })` hai. **Max length missing hai.**
* **Impact:** Koi malicious user ya bot API par hit karke 10MB ka lamba text bhej sakta hai. Is se tumhara database (MongoDB) jaldi bhar jayega, API response slow ho jayegi, aur server memory crash ho sakti hai.
* **Fix:** Fauran max limit add karo: `.isLength({ min: 10, max: 5000 })` (ya jo bhi tumhare platform ki limit hai).

**3. PostId Validation Optimization (Tech Debt)**

* **Issue:** Tumne `postId` ke liye `.isString()` use kiya hai aur comment mein likha hai ke MongoDB validation controller mein handle hogi.
* **Fix:** Express-validator pehle hi `isMongoId()` function provide karta hai. Tum apna code clean karne ke liye `.isMongoId().withMessage('Invalid Post ID format')` use kar sakte ho taake invalid requests controller tak pohnchne se pehle hi middleware mein drop ho jayein.

* **AI Availability on Edit (Feature Loop):** Hamara USP AI hai. Jab user `[Edit]` par click kare, toh usko wahi "AI Writing Assistant" aur "Get Suggestions" wale features wapas milne chahiye jo Create Post mein thay.

### 🏗️ 2. The Tabs Architecture (PM & Growth Strategy)

Tumne 4 tabs diye hain: `[Posts]`, `[Comments]`, `[Replies]`, `[Contributions]`. Ek naye user ke liye inme kaafi **Cognitive Confusion** hai.

* **Comments vs Replies:** Micro-blogging (Twitter/X style) mein "Comments" aur "Replies" ek hi cheez hoti hain. Alag alag tabs hone se user confuse hoga ke meri discussion kahan gayi.
* **Fix:** In dono ko merge karke sirf **"Replies"** ka tab banao. Yeh standard mental model hai.


* **The "Contributions" Tab:** Yeh term GitHub ya open-source platforms par use hoti hai. Ek social platform par "Contributions" ka kya matlab hai? Agar yeh guest posts ya articles hain, toh iska naam **"Articles"** ya **"Highlights"** hona chahiye taake aam user samajh sake.

### 🎨 3. Feed Rendering & Data Integrity (Frontend & Backend Bugs)

Maine tumhari feed ke data mein kuch specific bugs notice kiye hain:

* **Backend Validation Bug (Duplicate Tags):** Tumhari ek post ("India’s World Cup Highlights...") mein tags kuch is tarah render ho rahe hain: `*india* *india* *india* *worldcup*`.
* **Fix:** Backend (Node.js) post create/update karte waqt duplicate tags ko filter nahi kar raha. Backend controller mein `tags = [...new Set(tags)]` lazmi lagao taake database mein duplicates save na hon.


* **The "Wall of Text" (No Truncation):** Tumhari feed mein lambi posts poori ki poori render ho rahi hain bina cut hue. Agar ek user 500 words ki post karega, toh profile feed scroll karte karte user thak jayega.
* **Fix:** CSS/React mein `line-clamp` use karo (e.g., maximum 4 lines dikhao) aur uske baad **"Show more"** ka link do.


* **Missing Engagement Icons (UI):** Posts ke timestamps (e.g., "4 months ago") ke sath `*0* *0*` likha aa raha hai. Yeh likely Likes aur Comments ka count hai, lekin wahan icons missing hain. Numbers bina icons ke (❤️, 💬, 🔁) user ko samajh nahi aayenge.
